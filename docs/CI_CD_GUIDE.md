# CI/CD Guide — Bridge Creative Engine

Esta guía explica cómo configurar el pipeline de CI/CD con GitHub Actions para el proyecto Bridge Creative Engine.

## 📋 Estado actual

- ✅ El proyecto está desplegado en **Vercel** (con auto-deploy desde `main`)
- ✅ El **Cloudflare Worker** está desplegado manualmente
- ⚠️ **GitHub Actions workflow** aún no configurado (PAT actual sin scope `workflow`)

## 🎯 Objetivo del CI/CD

Cada `git push` a `main` debe ejecutar:
1. ✅ typecheck (TypeScript sin errores)
2. ✅ lint (ESLint sin warnings)
3. ✅ test (246+ unit tests)
4. ✅ build (Vite production build)
5. ✅ audit (GEMINI como segunda mano de validación)
6. ✅ comment (resumen en PR con coverage)

## 📝 Workflow completo (`.github/workflows/ci.yml`)

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

# Cancelar runs anteriores si hay un nuevo push
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  # === JOB 1: Validación rápida ===
  validate:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Typecheck
        run: pnpm typecheck

      - name: Lint
        run: pnpm lint

  # === JOB 2: Tests + Coverage ===
  test:
    needs: validate
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Run unit tests with coverage
        run: pnpm test:coverage
        env:
          CI: true

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          files: ./coverage/lcov.info
          fail_ci_if_error: false
        continue-on-error: true

      - name: Comment coverage on PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const summary = fs.readFileSync('./coverage/coverage-summary.json', 'utf8');
            const data = JSON.parse(summary);
            const lines = data.total.lines.pct;
            const branches = data.total.branches.pct;
            const functions = data.total.functions.pct;
            const body = `### 🧪 Test Coverage\n\n| Metric | % |\n|---|---|\n| Lines | **${lines.toFixed(2)}%** |\n| Branches | ${branches.toFixed(2)}% |\n| Functions | ${functions.toFixed(2)}% |\n\n> _Auto-generated by CI_`;
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body
            });

  # === JOB 3: Build production ===
  build:
    needs: validate
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build production
        run: pnpm build

      - name: Upload dist artifact
        uses: actions/upload-artifact@v4
        with:
          name: dist-${{ github.sha }}
          path: dist/
          retention-days: 7

  # === JOB 4: E2E tests (Playwright) ===
  e2e:
    needs: build
    runs-on: ubuntu-latest
    timeout-minutes: 20
    steps:
      - uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Install Playwright browsers
        run: pnpm exec playwright install --with-deps chromium

      - name: Build app (needed for preview)
        run: pnpm build

      - name: Run E2E tests
        run: pnpm test:e2e
        env:
          CI: true

      - name: Upload Playwright report on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 7

  # === JOB 5: Status check final ===
  all-passed:
    needs: [validate, test, build, e2e]
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - name: All checks passed ✅
        run: |
          echo "✅ typecheck: OK"
          echo "✅ lint: OK"
          echo "✅ tests: OK"
          echo "✅ build: OK"
          echo "✅ e2e: OK"
          echo ""
          echo "🚀 Ready to merge!"
```

## 🚀 Setup paso a paso

### Opción A: GitHub UI (recomendado, 2 min)

1. Ve a https://github.com/frank-vcorp/production_studio
2. Click **Add file** → **Create new file**
3. Path: `.github/workflows/ci.yml`
4. Pega el contenido del workflow de arriba
5. Click **Commit new file**
6. Espera ~1 min a que GitHub Actions ejecute el primer run

### Opción B: Nuevo PAT con scope `workflow` (5 min)

Si prefieres crear el archivo localmente y push:

1. https://github.com/settings/tokens/new
2. Name: `bridge-cd-workflow`
3. Scopes: ✅ `repo` + ✅ `workflow`
4. Generate token
5. Copia el token
6. Actualiza el remote:
   ```bash
   cd /mnt/Datos/Proyectos\ 2.0/production_studio
   git remote set-url origin https://<NEW_TOKEN>@github.com/frank-vcorp/production_studio.git
   git add .github/workflows/ci.yml
   git commit -m "ci: add GitHub Actions workflow"
   git push origin main
   ```

## 🐛 Troubleshooting

| Error | Solución |
|---|---|
| `pnpm: command not found` | El step `pnpm/action-setup@v4` debe ir antes de cualquier `run: pnpm` |
| `Playwright browsers not found` | `pnpm exec playwright install --with-deps chromium` (en job e2e) |
| `Worker secrets not available` | Para deploy del Worker desde CI, necesitas Cloudflare API token |
| `Vercel deploy fails` | Verifica que `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID` están en GitHub Secrets |

## 🔄 Deploy del Worker desde CI (opcional, avanzado)

Si quieres que CI también redespliegue el Worker en cada push a main, añade un job:

```yaml
  deploy-worker:
    needs: [validate, test, build, e2e]
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install worker deps
        run: cd worker && pnpm install --frozen-lockfile

      - name: Deploy to Cloudflare
        run: cd worker && pnpm exec wrangler deploy
        env:
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
```

Para configurarlo:
1. https://dash.cloudflare.com/profile/api-tokens
2. Create Token → Edit Cloudflare Workers template
3. Add el token a GitHub Secrets como `CLOUDFLARE_API_TOKEN`
4. Tu Account ID también como `CLOUDFLARE_ACCOUNT_ID`
