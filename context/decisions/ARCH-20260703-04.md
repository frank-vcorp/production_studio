**Autor:** INTEGRA

## 🎯 Contexto y Problema

**Problema Fundamental:** Los modelos de video generativo (Veo, Runway, Pika, Sora) en modo **Text-to-Video (T2V)** o **Image-to-Video (I2V) con prompt suelto** alucinan:
- Entornos que no existen (taller genérico vs tu taller real)
- Objetos/personas no provistas (mecánicos actores, herramientas inventadas)
- Iluminación/colores inconsistentes con la marca
- Discontinuidad visual entre clips (clip 1: taller azul, clip 2: taller naranja)

**Evidencia empírica:** En pruebas previas, prompts "cinemáticos" detallados sin ancla visual estricta producen videos "bonitos pero falsos" — no representan el negocio del cliente.

---

## 🏛️ Decisión: Keyframe Chain System (I2V Anclado Estricto)

### Principio Core

> **Cada clip de video es una transición I2V entre DOS keyframes ancladas a la realidad.**
> 
> - **Frame 0 (t=0)** = Keyframe IN: Imagen real subida por usuario **O** generada por Imagen 3 desde keyframe real anterior
> - **Frame Final (t=duration)** = Keyframe OUT: Imagen real siguiente **O** generada por Imagen 3 desde keyframe IN actual
> - **Veo SOLO interpola** entre estas dos anclas visuales conocidas
> - **Prompt incluye análisis visual exhaustivo** de AMBAS anclas + intención humana + specs cámara

## 📐 ARQUITECTURA DEL KEYFRAME CHAIN

### 1. Topología Fija (6 Keyframes Base + 4-5 Generadas)

```
TIMELINE VISUAL (27s total)

KF0: LOGO (Real)                    → Bumper I2I (3s) → KF1_IN
     │                                                                  │
     ▼                                                                  ▼
KF1_IN: PROBLEMA (Real)  ──I2V (4s)──▶  KF1_OUT: PROBLEMA AMPLIADO (Auto Imagen 3)
     │                                                                  │
     │                    ┌─────────────────────────────────────────┐   │
     │                    │  PROMPT VEO INCLUYE:                    │   │
     │                    │  • Análisis visual KF1_IN (Vision)      │   │
     │                    │  • Análisis visual KF1_OUT (Vision)     │   │
     │                    │  • Intención: "abrir a motor sucio"     │   │
     │                    │  • Cámara: macro→dolly out→steadicam    │   │
     │                    │  • Regla: "NO INVENTES entornos"        │   │
     │                    └─────────────────────────────────────────┘   │
     ▼                                                                  ▼
KF2_IN: TALLER (Real)     ──I2V (6s)──▶  KF2_OUT: DETALLE TALLER (Auto)
     ▼                                                                  ▼
KF3_IN: SOLUCIÓN (Real)   ──I2V (7s)──▶  KF3_OUT: MÉTRICA VISIBLE (Auto)
     ▼                                                                  ▼
KF4: CTA BASE (Render Canvas: Foto real + overlay marca) ──I2V (4s)──▶ KF5
     ▼                                                                  ▼
KF5: CTA FINAL (Real: Foto recepción)  ◀── FIN (3s)
```

### 2. Tipos de Keyframe

```typescript
// types/keyframe.ts
type KeyframeRole = 
  | 'bumper_start'      // KF0: Logo subido
  | 'atencion_in'       // KF1_IN: Foto problema real
  | 'atencion_out'      // KF1_OUT: Generada Imagen 3
  | 'interes_in'        // KF2_IN: Foto taller real
  | 'interes_out'       // KF2_OUT: Generada Imagen 3
  | 'deseo_in'          // KF3_IN: Foto solución real
  | 'deseo_out'         // KF3_OUT: Generada Imagen 3
  | 'accion_in'         // KF4: CTA Base (render canvas)
  | 'accion_out'        // KF4_OUT: Generada Imagen 3 (opcional)
  | 'cta_final';        // KF5: Foto recepción real

interface Keyframe {
  id: string;                    // kf_0, kf_atencion_in, kf_deseo_out_auto...
  role: KeyframeRole;
  label: string;                 // "Logo", "Problema", "Taller OUT (Auto)"
  source: 'user_upload' | 'generated_imagen3' | 'cta_render' | 'previous_output';
  blob?: Blob;
  base64?: string;
  mimeType?: string;
  
  // Análisis visual (Gemini Vision) — OBLIGATORIO para transiciones
  visualAnalysis?: VisualAnalysis;
  
  // Intención humana (usuario escribe)
  humanIntent?: string;          // "abrir a motor sucio"
  humanDescription?: string;     // "motor sucio desenfocado fondo"
  
  // Para generadas por Imagen 3
  generationPrompt?: string;
  
  // Specs cámara para transición OUT→IN siguiente
  cameraSpec?: CameraSpec;
  
  // Posición en timeline final (segundos)
  timestamp: number;
  
  // Estado
  status: 'empty' | 'uploaded' | 'analyzed' | 'generating' | 'generated' | 'approved' | 'failed';
}
```

### 3. Visual Analysis (Gemini Vision Output)

```typescript
interface VisualAnalysis {
  // Qué hay en la imagen (descripción técnica para prompt Veo)
  subject: string;              // "Manos guantes negros sosteniendo filtro aceite usado, grasa visible dedos"
  environment: string;          // "Motor diésel desenfocado, culata residuos aceite quemado, tonos marrón/negro"
  lighting: string;             // "Luz superior dura taller, sombras marcadas, reflejos especulares grasa"
  composition: string;          // "Primer plano extremo (macro), manos 70% encuadre, filtro centrado, DoF reducida"
  colorPalette: string[];       // ["#1a1a1a", "#2d1810", "#3d2914", "#8b4513", "#c0392b"]
  textures: string[];           // ["grasa viscosa", "metal cepillado", "papel filtro sucio"]
  cameraPosition: string;       // "Muy bajo, ángulo ligeramente contrapicado"
  depthOfField: 'shallow' | 'medium' | 'deep';
  dominantShapes: string[];     // ["círculo filtro", "rectángulo motor", "líneas herramienta"]
  technicalNotes: string;       // "Reflejos en grasa indican luz dura directa. No hay personas visibles."
  
  // Metadata
  analyzedAt: number;
  model: 'gemini-2.5-pro-vision';
  confidence: number;           // 0-1
}
```

### 4. Transiciones (El Corazón del Sistema)

```typescript
// types/transition.ts
interface KeyframeTransition {
  id: string;                          // trans_atencion, trans_interes...
  fromKeyframe: string;                // kf_atencion_in
  toKeyframe: string;                  // kf_atencion_out
  nodeKey: 'bumper' | 'atencion' | 'interes' | 'deseo' | 'accion' | 'cta';
  duration: number;                    // Segundos (3, 4, 6, 7, 4, 3)
  
  // Prompt COMPLETO aprobado por usuario
  prompt: string;
  
  // Camera spec para esta transición
  cameraSpec: CameraSpec;
  
  // Estado
  status: 'pending' | 'prompt_ready' | 'approved' | 'generating' | 'done' | 'failed';
  
  // Resultado
  videoBlob?: Blob;
  videoUrl?: string;
  veoOperationId?: string;
  generatedAt?: number;
  
  // Trazabilidad completa
  promptHistory: PromptVersion[];
}

interface PromptVersion {
  version: number;
  prompt: string;
  approvedAt: number;
  approvedBy: string;
  diffFromPrevious?: string;  // unified diff
  changeReason?: string;      // "Usuario cambió 'dolly out' por 'crane up'"
}
```

## 🔄 FLUJO DE GENERACIÓN (Pipeline Keyframe Chain)

### Paso 1: Usuario Sube Keyframes Reales (Mínimo 4)
```
Slots obligatorios:
├── KF0: Logo (PNG/SVG)                    → user_upload
├── KF1_IN: Problema (foto real)           → user_upload  
├── KF2_IN: Taller/Espacio (foto real)     → user_upload
├── KF3_IN: Solución/Resultado (foto real) → user_upload
├── KF4: CTA Background (foto real)        → user_upload (opcional, si no → gradiente marca)
└── KF5: CTA Final (foto recepción real)   → user_upload
```

### Paso 2: Análisis Visual Automático (Gemini Vision)
```
Para CADA keyframe real subida:
  1. Usuario click "Analizar" → llama Vision API
  2. Retorna VisualAnalysis JSON estructurado
  3. Usuario ve análisis → "Confirmar" o "Re-analizar"
  4. Status: 'analyzed'
```

### Paso 3: Generación Keyframes OUT Faltantes (Imagen 3)
```
Para CADA nodo AIDA sin OUT:
  1. Input: KF_IN (real) + VisualAnalysis + humanIntent + GlobalStylePrompt
  2. Prompt Imagen 3 construido:
     "CONTINUACIÓN DIRECTA de la imagen de referencia.
      MISMO entorno, MISMA iluminación, MISMOS materiales, MISMA paleta.
      Transformación: [humanIntent]
      Cámara: [cameraSpec para frame final]
      Estilo: [globalStylePrompt]"
  3. Imagen 3 genera KF_OUT candidata
  4. Usuario ve miniatura → "Aprobar" o "Regenerar"
  5. Vision Analysis automática en KF_OUT generada
  6. Status: 'approved'
```

### Paso 4: Prompt Approval Gates (Veo Transitions)
```
Para CADA transición (5 total):
  1. Sistema build prompt: buildKeyframeTransitionPrompt(from, to, node, camera, style)
  2. PromptGate UI: Textarea + miniaturas FROM/TO + diff vs original
  3. Usuario edita directamente → "APROBAR Y GENERAR"
  4. Prompt guardado en transition.prompt + promptHistory
  5. Status: 'approved' → listo para Veo
```

### Paso 5: Generación Paralela Veo (Background Job)
```
Job Queue inicia 5 workers paralelos:
  trans_atencion    → Veo I2V (KF1_IN → KF1_OUT)  4s
  trans_interes     → Veo I2V (KF2_IN → KF2_OUT)  6s
  trans_deseo       → Veo I2V (KF3_IN → KF3_OUT)  7s
  trans_accion      → Veo I2V (KF4 → KF5)         4s  (KF4 = CTA render, KF5 = foto real)
  trans_bumper      → Veo I2I (KF0 → KF1_IN)      3s  (Logo → Problema)

Cada worker:
  1. POST /api/gemini/generateVideo con prompt aprobado + input_image (base64 FROM)
  2. Poll operations cada 10s hasta done
  3. Descarga video → Blob → guarda en store.clips
  4. Actualiza transition.status = 'done'
```

### Paso 6: FFmpeg Smart Concat (Orden Garantizado)
```
Timeline fijo: [bumper, atencion, interes, deseo, accion, cta]
  1. Lee clips en orden fijo (no depende de orden generación)
  2. Concat demuxer → stream copy si codecs iguales
  3. Re-encode completo con LUT marca + color grading unificado
  4. Burn subtítulos (VTT generado desde voiceover timestamps)
  5. Mix audio (VO master + music bed opcional + ducking)
  5. Output: master.mp4 9:16 30fps ≤30s
```

## 🛡️ GARANTÍAS ANTI-ALUCINACIÓN (Checklist Técnico)

| Capa | Garantía | Implementación |
|------|----------|----------------|
| **Input** | Usuario provee realidad visual | 4-6 fotos reales obligatorias para generar |
| **Análisis** | IA describe SOLO lo que ve | Vision prompt: "NO infieras, NO inventes, NO completes" + temp 0.1 |
| **Prompt** | Anclas visuales inyectadas obligatoriamente | `buildKeyframeTransitionPrompt` incluye `formatVisualAnalysisForPrompt(fromVA, 'INICIAL')` + `formatVisualAnalysisForPrompt(toVA, 'FINAL')` |
| **Regla Explicita** | "NO INVENTES" en prompt | Texto literal: `⚠️ REGLA ABSOLUTA: USA LA IMAGEN DE REFERENCIA COMO BASE VISUAL INICIAL. NO GENERES ENTORNOS, OBJETOS, PERSONAS, TEXTURAS NI ILUMINACIÓN QUE NO ESTÉN PRESENTES EN AMBAS ANCLAS VISUALES.` |
| **Veo Call** | `input_image` obligatorio (I2V) | `generateVideo({ prompt, input_image: fromBase64, ... })` — nunca T2V |
| **Validación Usuario** | Usuario VE anclas + prompt ANTES de generar | `PromptApprovalGate` muestra miniaturas FROM/TO + prompt editable |
| **Post-Gen** | Comparación perceptual (opcional) | `pixelmatch` frame 0 vs FROM keyframe → alerta si diff > threshold |
| **Continuidad** | OUT N = IN N+1 (por diseño) | Keyframe chain topology fija: `kf_atencion_out` → `kf_interes_in` (misma imagen) |

## 🎯 CASOS EDGE Y MANEJO

| Escenario | Resolución |
|-----------|------------|
| **Usuario NO sube KF2_IN (Taller)** | Bloqueador: "Sube al menos 1 foto por nodo" — no genera transición Interés |
| **Usuario sube SOLO KF1_IN (Problema)** | Genera KF1_OUT auto (Imagen 3) → Transición Atención OK. Interés bloqueado hasta KF2_IN. |
| **Veo falla en trans_deseo (safety/cuota)** | Reintento 3x → Fallback: "Generar KF3_OUT con Imagen 3 + transición simple (zoom/pan)" → Usuario notificado |
| **Usuario quiere cambiar KF3_IN tras generar** | Regenera: KF3_IN nueva → Vision → KF3_OUT auto → trans_deseo + trans_accion (adyacentes) → Smart Concat |
| **Imagen 3 genera KF_OUT inconsistente** | Usuario ve miniatura → "Regenerar" (hasta 3 intentos) → Si persiste: "Subir foto real para este slot" |

## 📊 MÉTRICAS DE CALIDAD (Objetivos)

| Métrica | Target | Medición |
|---------|--------|----------|
| **Fidelidad Visual (Usuario)** | > 90% "El video parece mi negocio" | Encuesta post-export |
| **Continuidad (Frame Match)** | < 5% pixel diff frame 0 vs FROM keyframe | `pixelmatch` automatizado |
| **Alucinación Objetos** | 0 objetos no presentes en anclas | Auditoría visual manual (sample) |
| **Consistencia Color** | ΔE < 3 entre clips adyacentes | Colorimetry en frames clave |
| **Tasa Aprobación Prompt** | > 80% prompts aprobados sin edición | Telemetría `promptHistory` |
| **Regeneración Visual** | < 20% clips regenerados | Telemetría `transition.status` |
