## 🎯 Objetivo

Cerrar los **2 hallazgos críticos** que GEMINI marcó como bloqueantes para commitear S4:

- **H1 [CRÍTICO]**: ADR-04 violado — `SplitViewHost.tsx` callbacks eran placeholders (solo toast) sin invocar `approveTransitionPrompt`.
- **H2 [ALTO]**: `SmartConcat` worker solo concatenaba, faltaban subs burn + audio mix opcionales (SPEC 4.5).

## ✅ Cambios Aplicados

### H1 — SplitViewHost callbacks con ADR-04 Gate
**Archivo:** `src/components/generation/SplitViewHost.tsx` (reescrito)

- 4 callbacks (`handleRegenerateVisual`, `handleRegenerateVO`, `handleUpdateSubtitles`, `handleUpdateCameraSpec`) implementados con patrón:
  1. **Gate ADR-04** vía helper `checkApprovalGate()`: si `transition.status !== 'approved'` → toast error + early return (NO regenera).
  2. **Build nueva versión** del prompt o data.
  3. `approveTransitionPrompt(newVersion)` — registra en store.
  4. `versionHistory.recordVersion()` con `changeReason` trazable.
  5. `generateTransition()` para visual (Veo I2V) o `setState` directo para VO/subs/camera.
  6. try/catch con toasts de success/error.
- Header JSDoc documenta el gate para futuros maintainers.
- Marca de agua `IMPL-20260704-01` en JSDoc.

### H2 — SmartConcat con subs burn + audio mix opcionales
**Archivos modificados:**

- `src/services/smartConcat.ts`: 
  - Interface `SmartConcatInput` ya tenía `burnedSubs` y `musicBed` (definidos pero no usados).
  - Ahora se **pasan al worker** vía `ffmpegService.smartConcat()`.
  - Retorna nuevo campo `appliedFilters: { subs: boolean; music: boolean }` para auditoría UI.
- `src/services/ffmpeg.ts`: 
  - `SmartConcatOpts` extendida con `burnedSubs` y `musicBed` (Blob).
  - Conversión `Blob → ArrayBuffer` para cruzar `postMessage` structured-clone.
- `src/workers/ffmpeg.worker.ts`: 
  - `SmartConcutMsg.payload` con `burnedSubs?` y `musicBed?: ArrayBuffer`.
  - `execSmartConcat()` refactorizado con 4 escenarios (solo / subs+music / only subs / only music / ninguno).
  - Filter chain dinámico: `subtitles=` para video, `amix + afade` para audio.
  - `subs.vtt` y `music.wav` escritos a MEMFS condicionalmente.

### Tests Nuevos

  - H2 Test C: ambos=true cuando vienen los dos
  - H2 Test D: ambos=false cuando no vienen

### Riesgos / Desviaciones
- **Riesgo bajo:** El busy state se removió del SplitViewHost (cada tab maneja el suyo internamente via VisualTab/VOTab/SubsTab/CameraTab). Si en el futuro se necesita un busy state global para deshabilitar el modal entero, habría que re-agregar y pasarlo a SplitViewEditor. Aceptable porque los tabs ya tienen su propio spinner.
- **Riesgo bajo:** El H2 worker extension usa `-t 30` hard cap (límite total del master). Si un usuario tiene >30s de timeline, los clips finales se truncarán. Esto ya era el comportamiento previo, no se introdujo regresión.
- **Compatibilidad:** El wrapper `smartConcat` mantiene `appliedFilters` como opcional → no rompe callers existentes.
